# Data availability

## Tracked checkpoints

This branch stores only compact inputs and outputs needed to audit or restart the two workflows:

- donor-assignment tables for libraries `1_001`-`1_016`
- the submitted Figure 2D gene-set table and compact gene-level expression lookup
- JASPAR 2024 and HOCOMOCO v14 motif bundles
- hg38 consensus-peak, reference-gene, and reference-TSS annotation tables
- the label-transfer summary
- Figure 2D target-region tables, standardized region tables, cross-database consensus motif matrices, all-estimable model results, support-sensitivity tables, and primary plus extended family-context PDF/PNG files

## External inputs

Raw and large controlled inputs are not distributed through Git. The complete reconstruction expects:

```text
data/raw/atac_cellranger/1_001/outs/{peaks.bed,fragments.tsv.gz,fragments.tsv.gz.tbi,singlecell.csv}
...
data/raw/atac_cellranger/1_017/outs/{peaks.bed,fragments.tsv.gz,fragments.tsv.gz.tbi,singlecell.csv}
data/raw/reference/merged_QC_split_SCT_merged_harmony-batch_Dims22_slim.h5ad
data/raw/reference/Morocco_scRNA-seq.h5ad
data/raw/reference/10k_PBMC_Multiome_nextgem_Chromium_X_filtered_feature_bc_matrix.h5
```

Starting after Signac requantification instead requires the 17 `*_q_matrix.mtx`, `*_q_barcodes.tsv`, and `*_q_peaks.bed` triplets under `data/processed/atac_requantified/`.

## Generated large files

The following reproducible products remain local:

- `data/processed/atac_requantified/`
- `results/intermediate/`
- `results/models/`
- `results/final/tala_integrated_multivi_labeled.h5mu`

These files should be distributed through the study data repository or institutional object storage when binary-object access is required. Exact paths and staged entry points are described in `manuscript_companion_atac_tfbs_reproduction.md`.
